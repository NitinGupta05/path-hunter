What is PathHunter?

PathHunter Ultimate is an interactive pathfinding visualizer. It lets you see how classic graph algorithms explore a grid, step by step: which cells they visit, how they backtrack, and how they construct a path from Start to Target.

Algorithms you can explore

Breadth-First Search (BFS)

BFS explores the grid level by level. On this unweighted grid, the first time it reaches the target, it has found a shortest path in terms of number of steps.

  • Data structure: Queue (FIFO)
  • Guarantee: Shortest path if one exists
  • Use cases: Social networks, broadcasting, reachability

Depth-First Search (DFS)

DFS goes deep in one direction before backtracking. It is more about exploration than optimality. On the grid, DFS can produce long or zig-zag paths — this is expected behavior, not a bug.

  • Data structure: Stack (LIFO) / recursion
  • Guarantee: Does not ensure shortest path
  • Use cases: Maze generation, backtracking, cycle detection

A* (A-Star) Search

A* combines the cost so far from the start and a heuristic estimate of the remaining distance to the target. It focuses the search towards the goal instead of exploring everywhere evenly.

  • Score: F = G + H
  • Heuristic here: Manhattan distance
  • Use cases: Maps, games, robotics pathfinding

How to use the visualizer

  • Use Pointer to drag Start and Target nodes.
  • Use Pencil to draw walls and obstacles.
  • Use Eraser to remove walls.
  • Change Map themes to explore different visual contexts (Abstract, Midnight City, Deep Jungle, Coastal Route).
  • Select an algorithm and click Visualize to watch it in action.

PathHunter is built to help you see how algorithms think — which nodes they expand, how they trade off exploration vs. optimality, and what makes BFS, DFS, and A* behave so differently on the same grid.
